# AI-XR-Project

#  Mech-Raid XR

공간 컴퓨팅 기반 다중 사용자 협력 피트니스 보스 레이드 프로젝트입니다[cite: 3]. [cite_start]기존 VR/XR 환경의 HMD 착용으로 인한 멀미와 물리적 단절감을 극복하기 위해, 실제 공간에 디지털 요소를 투사하고 LiDAR 센서로 움직임을 정밀 추적하는 차세대 XR 플랫폼을 지향합니다.

##  프로젝트 개요

* **개발 목적:** MMRRPG의 핵심 요소인 ‘보스 레이드’ 기믹을 오프라인 다 인원 XR 환경에 융합하여, 스포츠와 게임의 경계를 허무는 협동형 액션 콘텐츠를 제시합니다.
* **기대 효과:** 몬스터와 광역 레이저를 피하기 위해 끊임없이 움직이며 자연스럽게 심폐지구력을 향상시키고, 기믹을 파훼하는 과정에서 팀워크와 공간 지각 능력을 극대화할 수 있습니다.
* **개발 기간:** 2026.6.29 ~ 2026.8.27 (총 9주).

##  핵심 페이즈 및 기믹

* **Phase 1 (몬스터 스웜 퇴치):** 바닥에 무작위로 생성되는 ‘가상 무기 활성화 구역’을 밟아 무기를 활성화한 뒤 다가오는 소형 기계 미니언 무리를 퇴치합니다.
* **Phase 2 (중간 보스):** 바닥에 생성되는 4가지 색상(빨강, 파랑, 노랑, 초록)의 원형 마커에 1부터 4까지의 숫자가 무작위로 표시되며, 4명의 플레이어가 순서대로 발판을 밟아 광역 공격을 회피해야 합니다.
* **Phase 3 (최종 보스):** 시스템이 무작위로 지정한 ‘시작 지점’과 ‘도착 지점’을 잇는 가상의 좁은 직선상에 모든 플레이어가 일렬로 도열하여 동적으로 생성되는 지뢰 함정을 피해야 합니다.

##  아키텍처 및 핵심 기술

* **프로젝션 및 클라이언트:** 다수의 프로젝터를 동기화하기 위해 언리얼 엔진 5(UE5)와 nDisplay 플러그인을 중점 활용하며, 왜곡 보정을 통해 완벽한 룸 스케일 공간을 창조합니다.
* **실시간 좌표 추적:** LiDAR 센서로 수집된 포인트 클라우드 데이터를 언리얼 데디케이티드/리슨 서버가 로컬에서 즉각 수신하고 칼만 필터로 보정하여 다중 플레이어의 좌표를 지연 없이 추적합니다.
* **서버 알고리즘 최적화:** 일렬 정렬 판별 시 부동소수점 오류나 Divide-by-Zero를 방지하기 위해 외적 기반의 삼각형 면적 공식을 활용한 언리얼 C++ 기반 판정 로직을 구현합니다.
* **생성형 AI 도입:** 7~8B 수준의 경량 오픈소스 LLM과 QLoRA 파인튜닝을 통해 플레이어의 실시간 행동 데이터를 분석하여 보스의 도발 및 지시 대사를 동적으로 생성합니다[cite: 17, 20, 21]. [cite_start]비동기 아키텍처(FastAPI) 분리와 시청각적 마스킹 기법을 통해 통신 지연을 완벽하게 은닉합니다.

##  팀 구성 및 역할

* **팀장 박건령:** 프로젝트 총괄, 서버 아키텍처 구축 및 게임 진행 제어, LiDAR 2D/3D 클라우드 좌표 데이터 로직 처리 및 공간 알고리즘 구현.
* **팀원 김시현:** 클라이언트 구현, 언리얼 엔진 기반 CAVE 및 nDisplay 프로젝션 환경 세팅, Meta Quest 3 프로토타이핑, 에셋 배치 및 시각 이펙트 최적화.

##  Ground Rules & 협업 컨벤션

안정적인 프로젝트 관리를 위해 아래의 규칙을 엄수합니다.

* **정기 회의:** 주 1회 온라인 진척을 공유하고 실증 테스트를 진행합니다.
* **사전 검증 원칙:** 리소스 낭비를 막기 위해 맵 스케일과 기믹 동선은 반드시 Meta Quest 3 환경에서 사전 검증합니다.
* **코드 리뷰:** 핵심 C++ 및 서버 코드는 PR 생성 후 1인 이상의 리뷰가 필수입니다.
* **에셋 관리:** 언리얼 블루프린트 및 맵 에셋 충돌 방지를 위해, 작업 전 반드시 Notion의 '에셋 점유 현황판'에 작업자를 명시한 뒤 수정합니다.
* **커밋 컨벤션:** `feat`, `fix`, `docs`, `asset:` 등 Conventional Commits 규칙을 적용합니다.
